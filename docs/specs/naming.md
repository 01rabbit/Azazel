# Naming Convention

Back to: [README](../../README.md) | [Product Map](../products/product-map.md)

## Canonical Names

- **Azazel System**
  - The doctrine, philosophy, naming, and architecture entry point for the system family.
- **Azazel-Edge Gateway**
  - Field-deployable edge SOC/NOC implementation of Azazel doctrine.
- **Azazel-Gadget Shield**
  - Portable tactical implementation for personal defense on untrusted Wi-Fi.
- **Cyber Scapegoat Gateway**
  - The defensive role concept: absorb, delay, redirect, and expose hostile behavior.

## Legacy Name Mapping

- `Azazel-Pi` -> `Azazel-Edge Gateway`
- `Azazel-Zero` -> `Azazel-Gadget Shield`

Use legacy names only when migration context is required.

## Naming Rule

Primary product naming format:

`Azazel-<Form> <Role>`

Examples:

- `Azazel-Edge Gateway`
- `Azazel-Gadget Shield`

## External Presentation Wording

Recommended wording for conference pages, repositories, and talks:

- "Azazel System is a cyber defense doctrine and tool family."
- "Azazel-Edge and Azazel-Gadget are concrete implementations of that doctrine."
- "Azazel applies delaying action to cyberspace through detect, decide, delay, redirect, and observe loops."

Avoid vague claims. Prefer clear operational verbs: detect, decide, delay, redirect, observe, buy time.
