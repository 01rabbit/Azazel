# Azazel Logo Banner Design Guidelines

This document defines the visual standard for all Azazel series banner artwork.

## Core Principles
- Every banner must be immediately recognizable as an Azazel product.
- Preserve the Azazel ram as the primary brand symbol.
- Communicate the product's mission, not just cybersecurity aesthetics.
- Favor readability at GitHub README scale.

## Shared Layout
1. AZ identifier.
2. Product name.
3. One-line mission statement.
4. Codename (optional).
5. Three-word value proposition.
6. Product-specific visual flow.
7. Capability footer.

## Shared Elements
- Dark background.
- Azazel ram emblem.
- Consistent spacing and hierarchy.
- Minimal readable text.

## Product Differentiation
- Knowledge: Deep crimson. Theme: Observe → Correlate → Advise.
- Fabric: Indigo/violet. Theme: Shared contracts and interoperability.
- Edge: Tactical amber/red. Theme: Deterministic decisions.
- Gadget: Cyan/green. Theme: Portable edge protection.

## Anti-patterns
- Do not create infographic-style banners.
- Avoid generic hacker imagery.
- Do not make the ram secondary.
- Avoid unreadable micro-text.
- Ensure visuals match README architecture.
